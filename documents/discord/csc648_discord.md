**Thread** 2022-12-20
> **Mark:** One big mistake that I see students doing in 648 often for the front end is that they tend to try to build everything from scratch.  There are LOTS of templates/tools available to you (e.g. bootstrap, mui, tailwind, etc).  You will save yourself a lot of grief if you use those resources.

**Thread** 2022-12-20
> **Mark:** I do not recommend learning docker for your 648 project either.  The docker ecosystem is set up in a way that requires you to learn how its networking works (e.g. how to get your front end to communicate with your back end and database server).

Some other useful knowledge would be to understand .env files for development and deployment.  This is not necessary, however, and can cause confusion with your team.  Our team did not use .env files and we just un/commented out the urls between development and deployment.

async function generateKeys() {
    const keyPair = await crypto.subtle.generateKey(
        {
            name: 'RSA-OAEP',
            modulusLength: 2048,
            publicExponent: new Uint8Array([1, 0, 1]),
            hash: 'SHA-256',
        },
        true,

    );

    const publicKey = await crypto.subtle.exportKey('spki', keyPair.publicKey);
    const privateKey = await crypto.subtle.exportKey('pkcs8', keyPair.privateKey);

    };
}
