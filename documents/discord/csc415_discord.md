  → Absolutely.  If you haven’t noticed yet, I will tell you to focus on code safety.  It is VERY easy to step all over memory allocations in C.  They are in no way safe.  You can read/write into pretty much any part of memory you are using intentionally or not.  A one digit memory offset error can have cascading effects in loops or multiple function calls.  In C, you never know where your data structure ends unless you decide to specifically track it.  Want a dynamically resizable structure?  You will not only have to create the structure for it, but you will have to program all the functions that handle the reading, writing, tracking, allocation, reallocation, and truncation of that structure.
