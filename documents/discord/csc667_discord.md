**Thread** [2021-01-25]
> nice, are the class projects you build in his class resume worthy?
  → The first project is you build a web server app in Java — the purpose of this is to teach you how HTTP req/response flow works at a base level, so moving on to web server apps like Express will make more sense because you understand the foundation. **Do not delay working on this**. The DAY that he assigns this, start designing/sketching your architecture. Every day, every hour counts with this project, because you'll end up doing *a LOT* of debugging and need all the time you can get. Plan the hell out of it before you start coding, so you always have a guide to follow and don't have to go back and re-code/re-build a bunch of stuff. The only other project is the term project, which is to build a multiplayer game online. Our group did poker, but you can probably save yourself a lot of headache and do something like Uno or Monopoly. Poker gets very complex very quickly when you start considering split pots and whatnot. Calculating the winner alone was a huge headache.
  → But both those projects are right now in my resume
